### Functional summary of what got built

- **Dialog hook**: `MarineOpsCMD` rules.csv command + `MarineOpsDialogDelegate`
  using `showCustomVisualDialog` (no forced buttons — mod owns dismiss).
- **Widget layer**: `Widget` / `BaseWidget` / `WidgetRoot` / `ButtonWidget`
  + raw LWJGL input routing, hit-testing.
- **Planet rendering**: `PlanetBillboardDrawable` (flat unwrap via Starsector's
  sprite GL texture) and `PlanetSphereDrawable` (UV sphere using the same
  equirectangular texture).
- **`BridgeRenderer` decoupled** from its hardcoded cube — accepts external
  scene + camera via `setScene`. Cube logic moved to `BridgePanelPlugin`.
- **Bitmap font system**: `BitmapFont` parses .fnt + draws glyph quads;
  `Fonts` constants; `LabelWidget`.
- **i18n**: `Strings.get` reads `mod/data/strings/strings.json` under a
  `marineOps` namespace. All UI text routes through it. Translation mods
  layer on top.
- **Three-column ops screen**: `ColumnLayout` data class, `OpsPanel` abstract
  base, `ClientListPanel` / `TacticalMapPanel` / `PlanetIntelPanel` siblings.
- **Planet intel block**: type, faction, population, conditions pulled from
  `planet.getMarket()`.
- **Client list**: planet's main faction + Independent + Pirates + other
  factions with system presence; HOSTILE+ locked with reason text; faction
  crests, rep-color-coded status. Selection state on `MarineOpsContext`.
- **Mission system**: `MissionType` / `RiskLevel` enums, `Mission` data,
  `MissionGenerator` with deterministic per-(planet, client) seed,
  `MissionNodeWidget` (colored circle + type glyph), `MissionPopupOverlay`
  (hover popup with name, type, risk, payout, requirements).

## Notes / accepted tradeoffs

- **Pole warping on planet sphere** — equirectangular UV-sphere singularity
  at poles. Accepted as-is. Cheapest mitigation is a fragment-shader
  fade-to-tint near poles; queued in backlog under Polish.
- **`ClientRowWidget` hover flicker on selection rebuild** — when the player
  picks a client, plugin rebuilds the entire widget tree (one-frame hover
  reset). Acceptable for now; per-panel `WidgetRoot` refactor would fix it.
- **`Fonts.VICTOR_10` removed** — UI minimum is Orbitron 20 after playtest
  feedback. Captured in [[ui-font-minimum]] memory.
- **Mission name generation** — hardcoded English templates in
  `MissionGenerator`, not yet i18n. Static UI labels are all translated;
  generated mission names will follow when a template system is needed.
