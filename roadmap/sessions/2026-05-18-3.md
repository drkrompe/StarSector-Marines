## Wall direction — derived → intrinsic mask

Pre-session, `BattleScreen` re-derived each wall cell's direction every
frame by querying four neighbor `GroundKind`s through an `isExteriorSide`
predicate. Two problems:

1. **The predicate only treated `INDOOR` as interior**, so commercial
   (TILE) and industrial (STRIPED) buildings classified their own
   floors as "exterior" to their walls. With `pickWallTile`'s
   `nExt ? 0 : (sExt ? 2 : 1)` short-circuit, that picked **north-cap
   art for south-perimeter walls** and west-cap for east — visible
   immediately in the commercial preview as the wrong cells.

2. **The whole model was wrong semantically.** A wall doesn't reorient
   itself because rubble appeared next to it. Wall direction is
   intrinsic to the wall — set when the structure was built, immutable
   under runtime mutations.

Landed in two passes. First (06cee3e) expanded `isExteriorSide`'s
interior set to `{INDOOR, TILE, STRIPED, RUBBLE}` so the previews
rendered correctly while the better design was being discussed. Then
(2d4ab9f) replaced the whole neighbor-derive approach with a per-cell
`byte[] wallDir` mask on `CellTopology`, set once by
`BuildingShellCore.stampPerimeterMask` from leaf geometry. The render
collapses to a pure mask lookup; `isExteriorSide` and the `GroundKind`
interior-set listing both go away.

`FortressWallStamper` (c6e87b2) got the same treatment via a single
final-pass sweep using the courtyard rect + `wallMask` to classify each
wall cell. Forward bunkers in the kill zone get the right outward-facing
bits automatically because their neighbors aren't in the courtyard.

## TILE = polished panel; SIDEWALK = outdoor brick

The current `TILE` ground kind was using `fl-tile-1..5` on
`Floors_Tiles.png` — brown brick that reads as **sidewalk slabs or roof
shingles**, not commercial interior. `BuildingCommercialFiller` made it
worse by mixing 20% random TILE accents into INDOOR floors, producing
"empty room with brown brick patches scattered."

Two-part fix:

- **TILE now means polished commercial floor** — `fl-2` (col 11, row 0
  on the road sheet). Single-cell, uniform across every TILE cell, same
  blanket-fill model `fl` uses for INDOOR. `BuildingCommercialFiller`
  drops the accent scatter; whole interior is TILE.
- **New `SIDEWALK` ground kind** — keeps the brown brick `fl-tile-1..5`
  art with the existing per-cell hash variant pool. `TrunkPlan.PRIMARY`
  boulevards, `PlazaFiller` plazas, and `DenseQuarterFiller`'s plaza
  ground all repoint here, so outdoor brick-paved pavement keeps its
  dedicated look. SIDEWALK renders through the floors-sheet draw path
  (16px cells); TILE renders through the road sheet (32px).

Schematic conquest preview now shows boulevards as distinct brown bands
separate from gray secondary streets — see
`build/map-previews/conquest-contact.png`.

## Doodad layouts — intentional fills replacing flat scatter

The pre-existing scatter was capped at 3 doodads per building regardless
of size, drawn randomly from a per-type pool. A 12×8 warehouse and a
5×5 hut both got the same 3 stray props. Buildings read as "empty room
with a chair," not as warehouse / shop / home.

New `BuildingLayouts` module with two primitives and four recipes:

**Primitives**
- `wallLine` — stamps a single prop along the inside of a wall at
  intervals, skipping cells within doorway clearance.
- `wallLineMix` — same but picks per cell from a variant array; gives
  hand-stacked / varied-stock feel.
- (`counterAtDoorway` — places a single prop just inside the first
  found entrance, leaving the cell directly inward of the door clear.)

**Recipes**
- `SHED` — TINY fallback (interior < 4 on either axis). Sparse scatter
  from the pool, 0-1 props.
- `HOME` — chair wall-line on one long wall (yellow/green alternating
  via `wallLineMix`) + a 1-2 chest cluster free-placed. Chairs +
  chests, not chairs + chairs.
- `SHOP` — shelf wall-line on both long walls (all 4 shelf variants
  per cell — empty / 1 / 2 / 3 — so stock reads as varied) + a
  counter-style desk at the doorway.
- `WAREHOUSE` — crate wall-line on both long walls (alternating box +
  crate variants) + a supervisor desk at the doorway.

Density scales naturally with interior area instead of capping at 3.
Doodads don't block pathing — they're visual + cover hints — so dense
rows are safe.

All eight `BuildingConfig` call sites (three direct fillers + the four
compound fillers) tagged with a recipe.

## Known gaps + iteration surface

Things flagged during the session but not landed — write into
`backlog.md` if they're not already there.

### Art

- **`COURTYARD` center (1, 1) and `fl-striped` center (7, 1) are blank**
  on `urban-tileset-2.png`. `pickStripedTile` substitutes the south-edge
  piece as a stand-in (see comment in `TileManifest.java:470`). Both
  need real center art. (Surfaced when the floor-candidates gallery
  rendered them as dark blue blocks.)
- **`SKYPORT_DOODADS` still has stale damaged refs** — `(7, 7)` is
  labeled "chest" but is `box-dam`. `DOODAD_POOL` and `WAREHOUSE_DOODADS`
  likely have similar mislabels. Audit and switch to clean variants.
- **`fl-tile-*` is now dormant** — earmarked for a future `ROOF`
  GroundKind (caves in, gains alpha when allied infantry enter).
- **Residential prop poverty** — no `bed`, `table`, `sofa`, `dresser`
  sprites. Until those exist, residential will read as "DMV waiting
  room with chairs and chests." Acceptable for now; eventual blocker
  on real "home" feel.

### Layout

- **Partition-aware recipes** — `BuildingShellCore` already adds
  optional interior partition walls. Could detect that and run
  **different recipes per room** — front-of-shop counter + back-of-shop
  storage = real auto-parts store. Today every room of a partitioned
  building runs the same recipe.
- **Intentional grate/striped shapes** — user's idea: a single grate in
  the middle of a striped warehouse reads as a designed feature, not
  noise. Needs a `centerShape(prop, dims)` primitive that stamps a
  fixed rectangle/cross inside the room.
- **More recipes** — OFFICE (desk+chair pairs in a grid), LAB,
  MESS_HALL. Primitives are reusable; mostly prop selection.
- **Pattern variety per recipe** — HOME always does wall-line + cluster.
  Across many buildings that repeats. Could roll between sub-patterns
  per building (e.g., "kitchenette in corner" vs "bed nook").

### Wall direction follow-ups

- **`isExteriorSide` deletion was clean** — no remaining callers in
  source. Doc reference in `roadmap/sessions/2026-05-18-2.md` is the
  only mention.
- **Damaged-wall 3×3 (cols 3..5, rows 4..6)** in urban-tileset is still
  unused. Future "wall HP < threshold" state could swap to it once the
  damaged variant is wired into the mask-based picker.
