## Slice 3a — Per-facing cover + RepositionToCover

**Story G.** Replaces the scalar-per-cell cover model with a directional
one: each cell has four cover values (N/E/S/W). A doodad contributes
isotropic cover to its own cell and per-facing cover to its four cardinal
neighbours, so a desk gives heavy cover *from the front* but not from
behind. `NavigationGrid.getCoverAt(x, y, fromDx, fromDy)` is the new
read API; `bestCoverCell` / `findFiringPosition*` use it.

**RepositionToCover** action — short-range cover-shift between bursts,
gated on a per-unit cooldown (`Unit.repositionCooldown`, 1.5s). Called
inline from `EngagePosture` after a shot, replacing the old 30% RNG
re-target. The `findFiringPositionCoverPreferred` filter is *strictly*
better — an MG already in heavy cover stays put rather than drifting
between same-cover cells.

## Slice 3b — Story A: Garrison ambush

**Overwatch posture** + `ENEMY_IN_KILL_ZONE` predicate. Garrison squads
(`Squad.holdsFireUntilKillZone`) sit on LOS but hold fire until the
kill-zone gate trips — enemy within `KILL_ZONE_RANGE_CELLS = 8` of any
squadmate, with LOS, sustained for `KILL_ZONE_LOS_TICKS_THRESHOLD = 6`
ticks (~0.2s, suppresses flicker on transient sightings). EngagePosture
gets a new precondition `ENEMY_IN_KILL_ZONE = true`; non-garrison squads
short-circuit the evaluator to true so the change is a no-op for marines
and patrols.

**BreakLOS** action — corner-duck via `findFallbackPosition` reusing
`Unit.fallbackCellX/Y`. Effect `UNDER_FIRE_AT_LOS = false`. Registered
but orphaned at this point — see RecoverFromAmbush below for the wire-up.

## Slice 3c — Story L: Choke-point ambush + Garrison cordon

**ChokePointHold** — instance-per-portal action, slots its squad onto
cells with LOS to the portal cell. Concentrated-fire trigger via
`ENEMY_IN_PORTAL_CELL(portalId)`. The first marine through eats the
whole squad's fire that tick.

**GarrisonCordon** — multi-portal degenerate. A squad garrisoned in a
room with more than one ingress can't all watch one doorway, so it
falls back to a fork of `HoldPortalCordon` minus the planter slot. One
holder per portal, facing outward.

**GarrisonAmbush** goal — MISSION-bucket, `customPlan` picks
ChokePointHold vs GarrisonCordon by ingress count, reads
`holdsFireUntilKillZone` for the hold-fire trigger.

## Slice 3.5 — Story M: Room breach + zone-mismatch

Inserted mid-slice from playtest: marines spawning inside a building
were engaged by a turret outside; the marine at the door took cover and
fought from the doorway one-by-one instead of the whole squad pushing
out so the rocket trooper could use their heavy weapon. The fix has two
halves — both are the same "your current zone isn't where the fight is"
signal stated differently.

**BreachAndAdvance** action — two-phase: (1) stack up just inside the
portal (slots `breacher:N`, arrival when ≥60% of slotted members are
within 1.5 cells of their stack-up cell, 2s timeout fail-open), (2)
move through the portal into the forward zone with a search-box.

**BreachToEngage** goal — ENGAGEMENT bucket. Picks the first
multi-cell zone past the squad's current zone (skipping 1-cell doorway
micro-zones), gives it as the forward zone to BreachAndAdvance. 5-deep
× 7-wide search box past the portal.

**Zone-mismatch target bias** in `TacticalScoring.findBestTarget` —
`TARGET_ZONE_MISMATCH_COST = 8f` softly penalises targets in a different
zone than the shooter. Soft gate (not hard exclude) because LoS through
a doorway is still a real shot if no closer target exists.

## AoE-spread fix

Playtest revealed marines clustering on a single cover cell (a desk in
the middle of an open field) — perfect AoE-rocket target. Two parts:

- `FIRING_OCCUPANCY_COST` bumped 4 → 8 — same-cell stacking penalty
  was undersized vs the new per-facing cover scale.
- New `FIRING_AOE_SPREAD_COST = 4f` over a radius-2 disc — counts allies'
  current cells OR pending path destinations. So a cell with 1 squadmate
  next door costs +4 even if the cell itself is open. Eager occupancy
  updates in `setPath` give same-tick claim visibility.

Out-of-band: building previews showed walls picking incorrect directional
tiles. Buildings flush against the map edge worked (OOB read as

## RecoverFromAmbush — BreakLOS closeout

End-of-session loose end. `BreakLOS` from Slice 3b was registered in
`INFANTRY_ACTIONS` but no goal desired `UNDER_FIRE_AT_LOS = false`, so
the planner never composed a plan using it.

**RecoverFromAmbush** sits in SURVIVAL with relevance 0.5 — outranks
ENGAGEMENT (caught-in-fire-lane → break LoS before re-engaging) but
loses to SurviveContact's 1.0 (broken morale → full retreat, not
duck-and-re-engage). desiredState = `UNDER_FIRE_AT_LOS = false`, no
customPlan — backward-chaining picks BreakLOS deterministically since
it's the only producer.

Squad-wide for now: every assigned member runs BreakLOS, and the action
naturally degrades for already-hidden members (`findFallbackPosition`
returns the member's own cell as the best distance-0 candidate, action
returns SUCCESS without movement). Per-member triggering ("only the
exposed members duck while the rest keep firing") is gated on Slice 4's
per-member action assignment.

## Notes / accepted tradeoffs

- **`BreakLOS` as a squad-wide step.** Per-member ducking would need
  Slice 4's per-member action assignment. The action degrades fine —
  unexposed members no-op — so this isn't a visible problem, but it's
  worth a note as we approach Slice 4.
- **`findFiringPositionCoverPreferred` returns null on no-improvement.**
  Strict-better filter — an MG in heavy cover stays cozy rather than
  drifting between same-cover cells. EngagePosture's call site checks
  for null and skips the move on a null return.
- **Zone-mismatch is a soft cost, not a hard exclude.** A target across
  a doorway is still a valid shot if no closer target exists. The
  `TARGET_ZONE_MISMATCH_COST = 8f` is enough to dominate ties but not
  to ignore a clean LoS shot through a wall opening.
- **`BreachAndAdvance` arrival uses 60% / 2s timeout.** Strict 100% would
  hang the squad on a single straggler (e.g. a wounded marine paused
  on a kit). The timeout fails open into the forward push so the squad
  never gets stuck staging.
- **`RecoverFromAmbush` relevance 0.5 vs `SurviveContact` 1.0.** Both in
  SURVIVAL; the picker (`Goal.pickMostRelevant`) takes max within the
  bucket. The relevance values are doing real work — they encode the
  semantic priority. Verified by an integration test that runs the
  picker directly.
- **`TileManifest.java` modification is unrelated work.** Showed up as
  modified throughout the session but it's a separate stream (floors
  sheet investigation per `BuildingZonePreviewTest` diff). Left
  untouched, will be committed separately.

## Personal note

The clean win this session was Slice 3.5 — Story M wasn't on the
roadmap; it came out of one playtest observation ("marines fought
one-by-one from a doorway instead of pushing out to engage") and what
felt at first like a separate problem ("the engage action should be
zone-aware") collapsed into the same fix. Same relevance gate, two
different framings. Worth remembering: when two ideas keep showing up
together, they're usually the same idea.

The AoE-spread fix is the other quiet win. The clustering bug was
sitting underneath the per-facing cover work the whole time —
`FIRING_OCCUPANCY_COST = 4` was tuned for scalar cover and just couldn't
push back against the new per-facing scoring scale. Bumping it plus a
separate spread cost (so cells *near* squadmates also penalise, not just
identical cells) is the right shape — "two marines on the same desk"
and "two marines on adjacent desks" are both AoE-rocket targets.

Slice 4 is the next big lift. Per-member action assignment is the
Stage 1 TODO that's been deferred since the planner landed — every
story that wants "team A fires while team B moves" needs it.
