## Notes / decisions made

- **JDK 25 `Random` LCG quirk.** `new Random(small_seed).nextFloat()`
  returns ~0.73 for all seeds 0–1000 — inherent LCG property, not a
  JDK regression. Production code never hits it because the map-gen
  RNG is shared across hundreds of calls. Fixed by making partition
  chance injectable (then replaced with the strategy pattern in
  Slice B).
- **`chamberIndex` on `PartitionLayout` (not `BuildingShellCore`).**
  Moved chamber-index computation onto the layout so callers don't
  need to know partition structure. Sorted-bisect on `int[] axes`
  generalizes cleanly from binary (1 axis) to ternary (2 axes).
- **Wall pair same-side-of-center constraint.** Both walls placed on
  one side of center (randomly chosen) so BFS-from-center always
  lands in an end chamber. Without this, the anchor can land in the
  middle chamber and distance-indexed labeling produces THRONE + INNER
  + INNER (no ENTRY).
- **Asymmetry is RNG-within-bounds.** The open question from the
  refactor doc resolved in favor of random wall placement within
  constraints. Visual variety across seeds while guaranteeing the
  three-chamber invariant.
- **Room-purpose overlay on conquest previews** is subtle at 5px/cell.
  The standalone diagnostic preview (`build/zone-previews/ternary-
  partition-labels.png`) is the better inspection tool. The conquest
  overlay is useful for confirming the wiring fires in context but
  won't pop visually until cell size increases.
